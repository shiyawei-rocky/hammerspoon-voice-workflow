# Reliability Scoring (Evidence-Driven)

This project uses a 0-10 reliability score generated by `scripts/reliability_score.py`.

## Inputs

- Transcript events: `~/.hammerspoon/transcripts/*.jsonl`
- Runtime log: `~/.hammerspoon/whisper.log`

## Score Composition

1. `completeness` (30%)
- Coverage of required actions (`Alt+E/A/C/W/Q/X/R/J`, `F5/F6/F7/F8`)
- Sample volume in the selected window

2. `stability` (35%)
- Penalizes runtime issues found in log:
  - `invalid UTF-8 code`
  - `normalize_spoken_numbers failed`
  - clipboard pollution
  - ASR/LLM request failures

3. `quality` (35%)
- Proxy checks on transcript outputs:
  - `Alt+E`: non-empty, no meta leakage
  - `Alt+A`: no markdown noise, <=4 sentences
  - `Alt+C`: either `未发现问题` or line-wise `类别:描述` format

## Interpretation

- `>= 9.0`: production-ready, low regression risk
- `8.0 - 8.9`: usable with minor risks
- `7.0 - 7.9`: medium risk, continue hardening
- `< 7.0`: not stable enough, require targeted fixes

## Usage

```bash
python3 ~/.hammerspoon/scripts/reliability_score.py --days 7
python3 ~/.hammerspoon/scripts/reliability_score.py --days 7 --json
```
